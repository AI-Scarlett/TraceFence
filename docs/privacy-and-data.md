# Privacy and Data

TraceFence is built to be useful without collecting more data than necessary.

## Summary

- core product data is processed locally
- the app does not need a hosted account for the default experience
- no advertising or analytics stack is required for core operation
- optional external services should be opt-in and disclosed clearly

## Local data

TraceFence may keep the following on the user's device:

- monitoring summaries
- session counts and token totals
- cached dashboard data
- preference settings
- folder or project selections
- license activation metadata if licensing is enabled

## Optional external integrations

If a user connects an external provider, TraceFence should clearly explain:

- what data leaves the device
- why it is sent
- which provider receives it
- how to disconnect it later

## Data retention

Stored data should remain available until the user clears it, resets the app, or uninstalls the product.

## Security posture

TraceFence should continue to rely on:

- least-privilege access
- local-only defaults
- explicit user permission for sensitive operations
- clear separation between product telemetry and customer content
